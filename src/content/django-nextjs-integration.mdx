# ادغام Django و Next.js — معماری، احراز هویت، SSR و Revalidate

راهنمای عملی ادغام Django (DRF) با Next.js: معماری، JWT/Session، SSR با کوکی، CORS، ISR و وب‌هوک برای بازسازی.

## پیش‌نیازها
- Node 18+، Next.js 14/15
- ساختار App Router
- دسترسی به دامنه و Search Console

## راهنمای سریع
- `metadataBase` را در `app/layout.tsx` تنظیم کنید.
- OG/Twitter با تصویر 1200×630 تنظیم شود.
- `alternates.canonical` را **مطلق** بنویسید.
- `app/robots.ts` و `app/sitemap.ts` را اضافه کنید.
- JSON-LD برای Organization/WebSite و صفحه‌ها.

## جزییات پیاده‌سازی

### نمونه‌ی متادیتا در layout
```tsx
export const metadata = {
  metadataBase: new URL("https://www.webdesignwithsina.ir"),
  title: { default: "WebDesignBySina", template: "%s | WebDesignBySina" },
  description: "SEO-first websites with Next.js + Django.",
  openGraph: {
    type: "website",
    url: "https://www.webdesignwithsina.ir",
    images: [{ url: "/og/cover.jpg", width: 1200, height: 630 }],
  },
  twitter: {
    card: "summary_large_image",
    images: ["/og/cover.jpg"],
  },
} as const
```

### canonical مطلق در صفحه‌ی داینامیک
```tsx
export async function generateMetadata({ params }): Promise<Metadata> {
  const url = `https://www.webdesignwithsina.ir/blog/${params.slug}`;
  return {
    alternates: { canonical: url },
  };
}
```

### robots و sitemap
- `app/robots.ts` و `app/sitemap.ts` را با دامنه‌ی اصلی تنظیم کنید.
- برای بلاگ‌های Markdown از `fs.stat` برای `lastModified` استفاده کنید.

## اشتباهات رایج
- تصویر OG با ابعاد/کنتراست نامناسب
- ندادن canonical مطلق در صفحات داینامیک
- بی‌توجهی به INP/CLS و جاگذاری نادرست اسکریپت‌ها
